# Standards and IP Policy

EDIForge must be useful for EDI work without redistributing restricted standards content.

## X12

X12 standards, implementation guides, and many trading-partner guides are copyrighted. EDIForge may include:

- Generic parser support for X12 syntax.
- Public-safe synthetic examples.
- User-provided schemas and maps.
- Community metadata that contributors have rights to publish.

EDIForge must not include:

- Paid X12 standards text.
- Proprietary implementation-guide content.
- Trading-partner maps without permission.
- Restricted code lists unless redistributable terms allow it.
- Derivative generated content from restricted materials.

Users are responsible for ensuring they have rights to use imported X12 guides, schemas, maps, code lists, and overlays.

## EDIFACT

UN/EDIFACT directories may be publicly available under terms that vary by source and use. Contributors must verify redistribution rights before adding EDIFACT metadata, labels, examples, or code lists.

## Bundled Schemas

Every bundled schema or mapping must include source and license metadata. When in doubt, keep examples synthetic and minimal.
