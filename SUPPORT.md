# Support

EDIForge is an open-source project. Public support happens through GitHub issues and discussions when enabled.

## Before Asking

- Check `README.md`, `docs/`, `CONTRIBUTING.md`, and existing issues.
- Try the latest release or the default branch when reporting behavior that may already be fixed.
- Reduce examples to the smallest synthetic EDI input, schema, or mapping that reproduces the question.

## Where to Ask

- Use a bug report for reproducible incorrect behavior.
- Use a feature request for new workflows, parser support, commands, endpoints, or UI behavior.
- Use a documentation issue for unclear or missing docs.
- Follow `SECURITY.md` for vulnerabilities or anything that could expose sensitive EDI data.

## Data Safety

Do not post real customer EDI, partner IDs, names, addresses, account identifiers, purchase orders, shipment numbers, invoice numbers, credentials, or proprietary implementation-guide content in public issues.

Synthetic samples are preferred. If redaction is necessary, make sure the redacted example still preserves the delimiters, segment order, and structure needed to understand the issue.
