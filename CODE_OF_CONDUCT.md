# Code of Conduct

EDIForge is a public open-source project. Participation should be direct, technical, and respectful.

## Expected Behavior

- Keep discussions focused on the project and its users.
- Critique code, designs, and tradeoffs without personal attacks.
- Use synthetic or safely redacted EDI examples in public.
- Respect licensing, standards IP, and contributor rights.
- Help maintain a useful environment for new and experienced contributors.

## Unacceptable Behavior

- Harassment, threats, hate speech, or targeted abuse.
- Publishing private contact information, credentials, customer EDI, partner identifiers, or other sensitive data.
- Submitting paid standards text, proprietary partner maps, restricted code lists, or data you do not have permission to share.
- Repeatedly derailing issues, pull requests, or discussions after maintainers ask for focus.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; restrict participation; or report abuse to GitHub when needed.

Security-sensitive reports should follow `SECURITY.md` rather than public discussion.
